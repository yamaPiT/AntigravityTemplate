# Antigravity 開発用ワークスペーステンプレート

このリポジトリ（フォルダ）は、Antigravity（愛称：ハル）を用いたAI主導のプログラム開発における標準的なワークスペーステンプレートです。

## 特徴

このテンプレートには、開発をスムーズに進め、品質を担保するための**AI開発基盤**と**参照資料**があらかじめセットアップされています。特に、最新の「V字モデル化 ADAプロセス」に対応したエージェントの専門スキルが組み込まれています。

* **Antigravity設定 (`.agents/`)**: 
    * **ワークフロー (`workflows/`)**: ADA（Agent-Driven Agile）プロセスのV字モデル標準開発手順が定義されています。
    * **スキル (`skills/`)**: 各開発工程を担当する専門エージェントと、ASDoQ文書品質モデルに基づき品質を保証する高次レビュアーエージェントのスキル（計8つ）が含まれます。
* **ドキュメント (`docs/`)**: 
    * ASDoQ文書品質モデルなど、開発・ドキュメント作成時に参照すべき資料が配置されています。
    * **`process/ada_document_guidelines.md`**: **【重要】** 通常の開発・壁打ち時にエージェントが優先参照する軽量なルールブックです。トークン節約と速度向上のため、通常はこのファイルに基づいてドキュメントが構成されます。
* **仕様書・設計書 (`doc/`)**:
    * 要求仕様書（SW105）、アーキテクチャ設計書（SW205）、およびソフトウェア適格性確認テスト仕様書・報告書（SW401）などの作成成果物はこちらのフォルダ下に保存されます。

## 本プロジェクト独自の運用ルール

* **トークン最適化と規格準拠のハイブリッド運用**:
    エージェントは通常、軽量なガイドライン（`ada_document_guidelines.md`）に基づきスピーディーに動作します。ただし、人間（マサ）から**「根本レビューをしてください」**という特定の指示があった場合に限り、`docs/process/` 下の重厚なPDF資料（ESPR, IEEE29148等）を全て読み込み、規格の精神に則った根本的な再構築・レビューを行います。これにより、コスト（トークン）の抑制と、専門的な品質維持を両立させています。
* **コード先行の絶対禁止**:
    いかなる場合も、まず要求仕様書（SW105）やアーキテクチャ設計書（SW205）を更新・合意してから実装に進みます。
* **詳細設計書の省略とコードの生きた仕様化**:
    管理の手間と仕様乖離を防ぐため、「詳細設計書」の作成は行いません。実装フェーズにおける「自律的実装計画」と、ソースコードへの「リッチコメント記述」が詳細設計の役割を担います。
* **適格性確認テストの完全自動化**:
    テスト工程の仕様設計と実行はAIが自動で行い、最終証跡として「ソフトウェア適格性確認テスト仕様書・報告書（SW401）」を自律生成します。
* **既存ソフトウェア資産がある場合の差分開発**:
    すでに要求仕様書・設計書・テスト仕様書などの開発資産がある場合、以下の手順でADAプロセスに取り込むことができます。
    1. 既存の各種ドキュメントをPDF化して `./docs/process/` フォルダに配置する。
    2. `./docs/process/ada_document_guidelines.md` の目次構造を、既存ドキュメントの目次構造に合わせて改訂する。
    3. 既存の要求仕様書・設計書・テスト仕様書を**Markdown形式**に変換し、`doc/` フォルダ下に配置する。
    4. 既存のソースファイルを `src/` フォルダ下に配置する。
    これにより、Antigravityが既存の文書体系を理解した上で差分開発を行えるようになります。

## 視覚化ツールの利用
* **Mermaid図の表示**: 
    VS CodeやCursorでMermaid図をグラフィック表示するには、拡張機能（**Markdown Preview Mermaid Support**）のインストールが必要です。AIは、この拡張機能で正しくレンダリングされるよう、安全で厳密な構文で描写します。

これらにより、ユーザーは純粋に「作りたい機能（要求）」をAIに伝え、高度な品質管理をAIに委ねながら、スピーディーな開発を実現できます。
