# Antigravity 開発用ワークスペーステンプレート

このリポジトリ（フォルダ）は、Antigravity（愛称：ハル）を用いたAI主導のプログラム開発における標準的なワークスペーステンプレートです。

## 特徴

このテンプレートには、開発をスムーズに進め、品質を担保するための**AI開発基盤**と**参照資料**があらかじめセットアップされています。特に、最新の「V字モデル化 ADAプロセス」に対応したエージェントの専門スキルが組み込まれています。

*   **Antigravity設定 (`.agents/`)**: 
    *   **ワークフロー (`workflows/`)**: ADA（Agent-Driven Agile）プロセスのV字モデル標準開発手順が定義されています。
    *   **スキル (`skills/`)**: 各開発工程を担当する専門エージェントと、ASDoQ文書品質モデルに基づき品質を保証する高次レビュアーエージェントのスキル（計8つ）が含まれます。
        *   **Implementers**: `requirements-engineer`, `architect`, `programmer`, `test-engineer`
        *   **Reviewers**: `requirements-reviewer`, `architecture-reviewer`, `code-reviewer`, `test-reviewer`
*   **ドキュメント (`docs/`)**: 
    *   ASDoQ文書品質モデルなど、開発・ドキュメント作成時に参照すべき資料が配置されています。
*   **仕様書・設計書 (`doc/`)**:
    *   要求仕様書（SW105）、アーキテクチャ設計書（SW205）、およびソフトウェア適格性確認テスト仕様書・報告書（SW401）などの作成成果物はこちらのフォルダ下に保存・管理されます。
*   ** Cursor ルール (`.cursor/rules/`)**:
    *   `project-rules.mdc` にて、コード先行の禁止や、人間とAIエージェントの明確な役割分担（V字モデルの実践）が定義されています。
*   **バージョン管理 (`.gitignore`)**:
    *   OSの一時ファイルやエディタ設定など、言語に依存しない汎用的な無視設定が含まれています。

## 使い方

このリポジトリはGitHubのテンプレート機能に対応しています。新しいプロジェクトを開始する際は、以下の手順でこのテンプレートを利用してください。

### ⚠️ カスタムルールにおける名称変更の注意
AIへの指示ルール（エディタのグローバルルール等）やドキュメント内には、初期設定としてユーザー名「マサ」、AIエージェント名「ハル」という固有の名称が記載されています。
他の方がこのライブラリをテンプレートとして利用される場合は、開発を開始する前に、これらの名称をご自身の名前やご自身のプロジェクトに適したAIの呼び名へと変更してください。

**名称変更・書き換えを行うべき主なファイル/フォルダ:**
*   `.cursor/rules/` フォルダ内の各ルールファイル（例: `project-rules.mdc`, `use-context7-for-docs.mdc`）
*   `.agents/skills/` フォルダ配下の各スキルの `SKILL.md` ファイル
*   `.agents/workflows/` フォルダ内の各ワークフロー定義ファイル（`.md`）
*   `docs/` フォルダ内の各種ドキュメント（例: `agent_development_guide.md`, `Implementation_plan.md`, `walkthrough.md`）
*   ご利用のエディタ設定における「AIへのシステムプロンプト（グローバルルール）」や「カスタムインストラクション」内の記述

### GitHubからプロジェクトを作成する場合（推奨）

1.  **テンプレートからリポジトリを作成**: GitHub上のこのリポジトリ画面右上にある緑色の **[Use this template]** ボタンをクリックし、**[Create a new repository]** を選択します。
2.  **リポジトリの作成**: 新しいリポジトリ名を任意に入力し、**[Create repository]** ボタンをクリックして作成します。
3.  **ローカルへのクローン**: 作成したご自身の新しいリポジトリをローカル環境にクローンします。

### ローカルで直接フォルダをコピーする場合

1.  **フォルダのコピー**: このテンプレートフォルダ一式をローカル環境でコピーし、フォルダ名を新しいプロジェクト名に変更します。

### 推奨される拡張機能

エディタ（VS Code / Cursor）での開発効率と視覚的な理解度を高めるため、以下の拡張機能のインストールを強く推奨します。

1.  **Markdown Preview Mermaid Support**
    *   要求仕様書や設計書内のMermaid図（フローチャート、シーケンス図等）をプレビュー画面でグラフィック表示するために必要です。
2.  **Japanese Language Pack for Visual Studio Code**
    *   エディタのインターフェースを日本語化し、操作性を向上させます。

### ワークスペースのセットアップ（共通）

1.  **ワークスペースを開く**: VS Code等のエディタで新しいプロジェクトフォルダを開き、Antigravity（ハル）とのセッションを開始します。
2.  **環境の初期化**: 開発するアプリケーションの要件（使用言語、フレームワークなど）をAntigravityに伝え、必要なディレクトリ（例：`src/`、パッケージ初期化等）の作成と環境構築を依頼してください。（例：「ReactとTypeScriptで新しいWebアプリを作るための初期設定をして」など）
3.  **.gitignore の更新**: 初期化された環境に合わせて、必要であればAntigravityに `.gitignore` へ言語やフレームワーク固有の設定を追加するよう依頼してください。


### 開発における自動化とワークフローについて

本テンプレート（やルール設定）の強力な特徴として、**ユーザー（人間）が明示的にワークフローを呼び出さなくても、AIエージェントが自律的に適切なプロセスを実行する**ように設計されています。

*   **ADAプロセスの暗黙的適用**: 
    開発時、「すぐにコードを書いて」と指示を出した場合でも、エージェントはコードの直接出力を拒否し、**デフォルトで（自動的に）ADAプロセスの規則に沿った開発（要求定義 → アーキテクチャ設計 → 実装 → テスト）**を行います。重厚なプロセスが必要な局面では、エージェント側から率先して `ADA-Process` を開始するかどうかを提案します。
*   **動的な要求とアーキテクチャ仕様の構築**:
    要求定義および設計フェーズでは、エージェントが固定のテンプレートに縛られず、参考ドキュメント（`docs/process/` 内）を読み込んだ上で、プロジェクト固有の技術領域（Web、組込み、AIなど）に最適な構成のドキュメントを自律的に立案します。
*   **テキストベース図表による視覚化と互換性確保**:
    生成される各種ドキュメントには、人間の理解を助けるために「Markdownの表」や「Mermaidによるフロー・アーキテクチャ図」が積極的に挿入されます。
    - **【💡プレビューに関する重要事項】** VS CodeやCursorの標準プレビューアでMermaid図をグラフィック表示するには、拡張機能（**Markdown Preview Mermaid Support**）のインストールが必要です。AIは、この拡張機能で正しくレンダリングされるよう、安全で厳密な構文（特殊文字のクォート囲み、HTML不使用など）で描写するようルールづけられています。もしコードブロックのまま表示されてしまった場合は、エージェントに「プレビューでエラーになっているから、安全な構文で書き直して」と指示してください。
*   **詳細設計書の省略とコードの生きた仕様化**:
    管理の手間と仕様乖離を防ぐため、「詳細設計書」の作成は行いません。実装フェーズにおける「自律的実装計画」と、ソースコードへの「リッチコメント記述」が詳細設計の役割を担います。設計ドキュメントとして合意・共有するのは「ソフトウェア要求仕様書」と「ソフトウェアアーキテクチャ設計書」のみです。
*   **適格性確認テストの完全自動化と人間の評価ループ**:
    すべてのテスト工程（単体・結合・適格性確認）の仕様設計と実行はAIが完全に自動で行い、最終証跡として「ソフトウェア適格性確認テスト仕様書・報告書（SW401）」を `./doc/` フォルダ下に自律生成します。人間はすべての処理が終わった後にこの報告書を確認してプログラムを評価し、必要に応じて要求を改訂（プロセス再始動）するという高度なフィードバックループに専念できます。

これらにより、ユーザーは純粋に「作りたい機能（要求）」をAIに伝え、最終的な「評価・改訂」を行うことにのみ集中できます。

##  context7 (MCPサーバー) の設定について

このリポジトリでは、`context7` というAIがライブラリを使うときに最新のドキュメントを参照できるようにするMCPサーバーを用いています。具体的には、次の設定をしています。設定が難しい場合は、`.\.cursor\rules\use-context7-for-docs.mdc` ファイルを削除してください。

### (1) context7 API Keyの取得
* [https://context7.com/](https://context7.com/) にGoogleアカウントなどでサインインする。
* 上部の `More...` メニュー内に `Create API Key` があるので、そこでAPI Keyを作成してコピーする。

### (2) AntigravityでのMCPサーバー設定
* Antigravity画面右上の三点ドットから `MCP Servers` を選択し、`View raw config` を選択する。
* 以下のように作成または追記する。`YOUR_API_KEY` に、先ほど取得したキーを入力する。

```json
{
  "mcpServers": {
    "context7": {
      "command": "npx",
      "args": ["-y", "@upstash/context7-mcp", "--api-key", "YOUR_API_KEY"]
    }
  }
}
```
